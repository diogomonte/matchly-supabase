# matchly

Match players for tennis and padel — powered by Supabase.

See @README.md for full setup guide and @package.json for all available scripts.

## Architecture

| Path | Purpose |
|---|---|
| `src/lib/supabase.ts` | Typed Supabase client + auth helpers |
| `src/types/supabase.ts` | **Auto-generated** DB types — never edit by hand |
| `supabase/migrations/` | Versioned SQL migrations, applied in timestamp order |
| `supabase/seed.sql` | Local dev seed data only, never migration logic |
| `supabase/config.toml` | Local Supabase config (ports, auth, storage) |

## TypeScript conventions

- ES modules only (`import/export`), never CommonJS (`require`)
- Import types with `import type` when importing type-only symbols
- All Supabase client calls must use the typed `Database` generic from `src/types/supabase.ts`
- Always destructure `{ data, error }` from Supabase calls and check `error` before using `data`
- No additional dependencies without explicit approval — the only runtime dep is `@supabase/supabase-js`

## Database conventions

- New migrations go in `supabase/migrations/` with timestamp prefix: `YYYYMMDDHHMMSS_description.sql`
- Use `supabase db diff` to auto-generate migration files; hand-write only when needed
- Every new table MUST enable RLS: `ALTER TABLE public.table_name ENABLE ROW LEVEL SECURITY;`
- Define SELECT, INSERT, UPDATE, DELETE policies for every new table
- Seed data goes in `supabase/seed.sql`, not in migration files

## Key workflows

- After ANY schema change: run `npm run supabase:generate-types` to sync `src/types/supabase.ts`
- To reset local DB and reapply all migrations: `npm run supabase:reset`
- To push migrations to hosted Supabase: `npm run supabase:migrate`
- Local Supabase dashboard: http://localhost:54323 (when `npm run supabase:start` is running)

## IMPORTANT rules

- NEVER modify `src/types/supabase.ts` directly — it is always regenerated by `supabase gen types`
- NEVER commit `.env` — credentials must stay out of version control (`.gitignore` already covers this)
- ALWAYS run `npm run supabase:generate-types` after any migration that changes the public schema
